# Manifesto

Everything I Understand, I Can Build.

This is the operating rule of the lab.

The complete operating protocol lives in `OPERATING_SYSTEM.md`.

Understanding is not knowing the right words. It is not collecting references. It is not calling an API. It is not repeating somebody else's explanation.

Understanding means I can:

1. rebuild the mechanism at low complexity;
2. recognize it inside a real high-complexity system;
3. debug it when it fails;
4. align it against a real implementation or source;
5. compress it into principles;
6. teach it so another person gains the model.

## Method

Every lab follows the same loop:

```text
see the black box
-> shrink it
-> build it
-> test it
-> compare it
-> record failures
-> compress principles
-> teach it
```

The smallest build is not a toy. It is a microscope.

## Anti-Patterns

- Reading without building.
- Bookmarking without testing.
- Calling a tool and mistaking usage for understanding.
- Explaining with vague abstractions before a working mechanism exists.
- Adding many features before the core loop is observable.
- Chasing the newest system without extracting transferable principles.

## North Star

When I truly understand something, I should be able to make a small version of it come alive.
